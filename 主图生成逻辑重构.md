# Plan: Multi-Agent 流程图生成系统

## Context

当前 `analyzeOverview()` 是单次 LLM 调用，只能看到入口函数源码 + 签名，无法深入理解被调函数的语义。对于复杂项目（如 FastAPI 应用），入口函数可能只是脚手架代码，业务逻辑藏在更深层。

新方案：**Lead Agent + Worker Agents 协作**。Lead 用纯 AST 找到核心业务函数，分发 Workers 并发语义化被调函数，结果汇入知识库，最终 Lead 基于核心函数源码 + 知识库生成流程图。

API 契约不变：`POST /api/analyze/overview` 仍返回 `FlowData {nodes, edges}`，前端无需改动。

---

## 一、找到核心业务函数（业务密度算法）

**新文件**: `backend/services/business_density.py`

纯 AST 分析，零 LLM 调用，毫秒级。

### 评分指标

| 指标     | AST 节点                                             | 权重      | 含义            |
| ------ | -------------------------------------------------- | ------- | ------------- |
| 控制流    | `If/For/While/Try/With` 数量                         | ×3      | 有分支/循环 = 在做逻辑 |
| 数据链    | `变量 = func()` 且变量被后续使用                             | ×2      | 有数据流转 = 在处理业务 |
| 领域调用   | 调用项目内函数（非基础设施）                                     | ×1      | 在调用业务函数       |
| 基础设施调用 | `include_router/add_middleware/setup/init/print` 等 | ×(-0.5) | 在搭架子，扣分       |

```
score = control_flow * 3.0 + data_chain * 2.0 + domain_calls * 1.0 - infra_calls * 0.5
```

阈值 `DENSITY_THRESHOLD = 5.0`，高于此值认为是业务函数。

### 算法流程

```
find_key_function(graph, project_files):
  1. 对每个 entry_point 计算密度分数
  2. 如果最高分 >= 阈值 → 返回该函数（如 TestProject 的 main）
  3. 如果全部低于阈值 → 下钻：取 entry 的 callees，重新评分
  4. 最多下钻 2 层，返回任意层级中得分最高的函数
  5. Fallback：都不够高则返回得分最高的 entry
```

### 两类项目的走法

**TestProject**：`main()` 有 if/for、数据链（token → get_user）、领域调用（register/login）→ 分数 ~15，直接命中。

**FastAPI 项目**：`main.py` 只有 `include_router/add_middleware` → 分数 ~-2.5，下钻到 `@router.post` handler → handler 有 if/await/数据链 → 命中。

## 二、知识库

**新文件**: `backend/services/knowledge_base.py`

### 选型：纯内存 dict

理由：asyncio 单线程，await 之间不会交错写入，无需锁。每次请求新建一个实例，请求结束即 GC，不需要持久化。

### 数据结构

```
@dataclass
class FunctionSummary:
    qualified_name: str    # "services/auth_service.py::AuthService.register"
    file: str
    line_start: int
    line_end: int
    summary: str           # "验证邮箱和密码格式，创建 User 实例存入内存字典"
    calls: list[str]       # 它调用的函数 qualified_names
    params: list[str]
    kind: str              # function/method/class

class KnowledgeBase:
    _entries: dict[str, FunctionSummary]

    def put(self, summary: FunctionSummary) -> None
    def get(self, qname: str) -> FunctionSummary | None
    def all_summaries(self) -> list[FunctionSummary]
    def is_complete(self, expected: set[str]) -> bool
    def format_for_prompt(self) -> str
        # "- register(username, email, password): 验证邮箱和密码格式，创建用户"
        # "- login(username, password): 校验凭据，生成 token"
```

### 3.3 消息传递 — 邮箱模式

**新文件**: `backend/services/agents/mailbox.py`

Agent 间通过 Mailbox 解耦通信，不直接调用彼此的函数。

```
@dataclass
class Message:
    sender: str           # "lead" / "worker-0" / "worker-1"
    receiver: str         # 目标 agent 名称
    msg_type: str         # "task" / "result" / "error"
    payload: dict         # 消息体

class Mailbox:
    """异步邮箱，每个 agent 有独立收件箱，exactly-once 消费。"""

    def __init__(self) -> None:
        self._boxes: dict[str, asyncio.Queue] = {}

    def register(self, agent_name: str) -> None:
        self._boxes[agent_name] = asyncio.Queue()

    async def send(self, msg: Message) -> None:
        box = self._boxes.get(msg.receiver)
        if box:
            await box.put(msg)

    async def read_inbox(self, agent_name: str) -> Message:
        """阻塞等待，读即消费（exactly-once）。"""
        return await self._boxes[agent_name].get()

    async def broadcast(self, sender: str, msg_type: str, payload: dict) -> None:
        for name, box in self._boxes.items():
            if name != sender:
                await box.put(Message(sender, name, msg_type, payload))
```

**通信流程**：

```
Lead                         Mailbox                      Worker-0, Worker-1, ...
  │                            │                              │
  ├─ send(task: "分析 register") ──→ worker-0 收件箱            │
  ├─ send(task: "分析 login")   ──→ worker-1 收件箱            │
  │                            │                              │
  │                            │    ←── read_inbox("worker-0") ┤
  │                            │         Worker-0 拿到任务      │
  │                            │         LLM 语义化             │
  │                            │         写入知识库             │
  │                            │    ──→ send(result → "lead")  │
  │                            │                              │
  ├── read_inbox("lead") ←─────┤        收到 worker-0 结果     │
  ├── read_inbox("lead") ←─────┤        收到 worker-1 结果     │
  │                            │                              │
  ├── 所有结果收齐               │                              │
  ├── 读知识库，生成流程图        │                              │
```

**Worker 生命周期**：

Worker 自己读源码、自己写 KB、通过 Mailbox 通知 Lead "完成了"。

**Mailbox 只传"任务"和"完成通知"，不传数据。Lead 最后从 KB 读结果。**

```
async def worker_loop(name: str, mailbox: Mailbox, graph, project_files, kb):
    while True:
        msg = await mailbox.read_inbox(name)
        if msg.msg_type == "shutdown":
            break
        if msg.msg_type == "task":
            qname = msg.payload["qualified_name"]
            # Worker 自己从 project_files 读源码
            defn = graph.definitions[qname]
            source = project_files[defn.file]
            func_code = extract_lines(source, defn.line_start, defn.line_end)
            # LLM 语义化
            summary_text = await call_qwen(...)
            # 写入 KB
            kb.put(FunctionSummary(qname, summary_text, ...))
            # 通过 Mailbox 通知 Lead "我完成了"
            await mailbox.send(Message(
                sender=name,
                receiver="lead",
                msg_type="done",
                payload={"qualified_name": qname}
            ))
```

**Lead 等待逻辑**：

```
# 分发任务
for i, qname in enumerate(needed):
    worker_name = f"worker-{i}"
    mailbox.register(worker_name)
    asyncio.create_task(worker_loop(worker_name, mailbox, ...))
    await mailbox.send(Message("lead", worker_name, "task", {"qualified_name": qname}))

# 等待所有完成通知
for _ in range(len(needed)):
    done_msg = await mailbox.read_inbox("lead")  # 只是通知，不含数据

# 全部完成后，从 KB 读取结果生成流程图
prompt_context = kb.format_for_prompt()

# 关闭 Workers
for i in range(len(needed)):
    await mailbox.send(Message("lead", f"worker-{i}", "shutdown", {}))
```

**为什么用邮箱而不是直接调用**：

- 解耦：Worker 不知道 Lead 的存在，只知道 Mailbox
- 可扩展：未来可替换为 Redis Streams / 消息队列，接口不变
- 可观测：所有消息经过 Mailbox，便于日志/调试
- 与 AgentForge 的 InMemoryMailbox 设计一致
