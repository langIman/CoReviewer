# 2026-04-10 开发日志

## 1. AST 数据持久化（内存缓存 → SQLite）

之前 ProjectAST（符号定义、调用关系、模块依赖）仅存在内存全局变量中，进程重启即丢失，与摘要（已持久化到 SQLite）不一致。

- `backend/dao/database.py`: `init_db()` 新增 3 张表：
  - `symbols` — 函数/类/方法定义（PK: qualified_name + project_name）
  - `call_edges` — 函数调用关系（自增 ID）
  - `modules` — 模块级信息及 import 依赖（PK: path + project_name）
- `backend/dao/ast_store.py`: 新建 AST 持久化 DAO
  - `save_project_ast()` — 清旧数据 + 批量写入三张表
  - `load_project_ast()` — 从三张表重建 ProjectAST 对象
  - `clear_project_ast()` / `has_project_ast()`
  - list 字段（decorators, params, imports）用 `json.dumps/loads` 序列化
- `backend/utils/analysis/ast_service.py`: `get_or_build_ast()` 改为三级查找：
  1. 内存缓存命中 → 直接返回
  2. 内存未命中 → 从 SQLite 加载 → 写入内存缓存 → 返回
  3. SQLite 也没有 → 重新构建 → 同时写入 SQLite 和内存缓存

`graph_cache.py` 保持不变，继续作为运行时一级缓存。entry_points 不单独建表，从 `symbols WHERE is_entry = 1` 查询。

## 2. Agent Tool 体系：5 个数据访问工具

为 Agent 的 tool-use 主循环提供原子化的数据访问能力。全部继承 `BaseTool`，通过 OpenAI function calling 格式注册。

| Tool | 文件 | 数据源 | 参数 |
|------|------|--------|------|
| `GetSummariesTool` | `tools/get_summaries.py` | SQLite summaries 表 | `summary_type` (file/folder/project)，必填 |
| `GetSymbolsTool` | `tools/get_symbols.py` | SQLite symbols 表 | `file`, `kind`，可选 |
| `GetCallEdgesTool` | `tools/get_call_edges.py` | SQLite call_edges 表 | `caller`, `callee`，可选 |
| `GetModulesTool` | `tools/get_modules.py` | SQLite modules 表 | `path`，可选 |
| `GetFileContentTool` | `tools/get_file_content.py` | 内存 file_store | `path`，必填 |

设计决策：
- **5 个 Tool 而非 1 个**：每个 Tool 参数结构不同，拆开后 LLM 更容易理解和正确调用
- 路径不存在时 `GetFileContentTool` 返回可用文件列表，方便 Agent 自纠正
- `tools/__init__.py` 统一导出所有 Tool
