# 2026-04-08 开发日志

## 1. 多文件类型支持

项目上传从仅支持 `.py` 扩展为支持 30 种文件类型（代码、配置、文档）。

- `backend/config.py`: `ALLOWED_EXTENSIONS` 扩展，新增 `AST_EXTENSIONS` 和 `is_ast_file()` 供 AST 管线专用
- `call_graph.py`、`entry_detector.py`: AST 过滤改用 `is_ast_file()`
- `file_service.py`: 错误信息泛化
- 前端: 移除 `accept=".py"` 限制，更新 i18n 文案

核心思路：上传层放宽文件类型，AST 静态分析（调用图、入口检测）仍仅针对 Python，非 Python 文件通过 LLM 摘要的 fallback 路径处理。

## 2. 项目上传后自动生成层级摘要

之前上传项目后调用的是旧 `generateProjectSummary()`（结果不展示），新的 `generateHierarchicalSummary()`（文件→文件夹→项目）需要手动点按钮。

- `UploadBar.tsx`: 上传成功后自动调用 `handleOverview()`，AIPanel 立即展示 loading 卡片
- `App.tsx`: 拖拽上传同步改造
- `handleOverview` 完成后设置 `summaryReady`，按钮正常出现

## 3. 基于摘要 + 调用图的模块划分

新功能：摘要完成后用户点击"模块划分"按钮，LLM 基于文件夹摘要（主）和 import 依赖（辅）将项目拆分为逻辑模块。

新增文件：
- `backend/services/module_service.py` — 读取摘要 + AST 依赖，调用 LLM，后处理确保全覆盖
- `backend/services/llm/prompts/module_prompts.py` — 模块划分 prompt，显式列出所有待分配路径
- `backend/controllers/module_controller.py` — `POST /api/module/split`
- `backend/dao/summary_store.py` 新增 `get_summaries_by_type()` 查询函数

前端：
- `api.ts` 新增 `splitModules()`
- `UploadBar.tsx` 新增橙色"模块划分"按钮
- `ResponseCard.tsx` 新增 `ModuleCards` 组件（可折叠卡片列表）

后续迭代修复了覆盖不全问题：prompt 中同时包含文件夹摘要和根目录散文件摘要，并显式列出所有路径；后处理 `_ensure_full_coverage()` 将遗漏路径按前缀匹配分配到最近模块。
