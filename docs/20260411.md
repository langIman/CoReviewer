# 2026-04-11 开发日志

## 新增：统一多语言 AST 解析（tree-sitter）

之前项目仅支持 Python 的 AST 静态分析（基于内置 `ast` 模块），Rust 等语言的文件上传后会被跳过。本次新增基于 tree-sitter 的统一多语言解析引擎，首先支持 Rust，架构可扩展到任意 tree-sitter 支持的语言。

### 核心设计

配置驱动的统一解析器，每种语言只需一个 `LangConfig` 配置字典，不需要单独的 parser 文件。Python 保持原有 `ast` 模块不变，其他语言走 tree-sitter 统一路径。

**数据契约不变**：无论解析什么语言，产出的 `ProjectAST` / `SymbolDef` / `CallEdge` / `ModuleNode` 与 Python 完全一致，下游服务（数据库、缓存、Agent tools、前端）零改动。

## 重构：项目初始化服务解耦

原先 `file_service.py` 混杂了文件上传、缓存失效、AST 构建等职责。提炼出独立的 `init_service.py`，职责单一。

| 文件 | 改动 |
|------|------|
| **`backend/services/init_service.py`** | **新建**：`initialize_project(project_name)` — 清理旧状态 → AST 构建 → 持久化 |
| `backend/services/file_service.py` | 去除 `invalidate_cache()` 和 AST 构建逻辑，改为调用 `initialize_project()` |

调用关系：
```
upload_project_files()          [file_service — 只管文件验证和存储]
  └─ store_project()
  └─ initialize_project()       [init_service — 只管初始化]
       ├─ invalidate_cache()
       ├─ clear_project_ast()
       ├─ clear_project_summaries()
       └─ get_or_build_ast()    [构建调用图 + 入口检测 + 持久化 SQLite]
```

### 未改动

- `dao/` 全部 — 存储层语言无关
- `agent/tools/` — 查询层语言无关

### ts_parser.py 结构

```
LangConfig          — 语言配置 dataclass（节点类型映射 + 回调函数）
_REGISTRY           — 语言注册表
通用 Walker:
  ts_extract_definitions()  — 从 CST 提取 SymbolDef
  ts_extract_calls()        — 从函数体提取 CallEdge
  ts_build_import_name_map() — 导入名 → qualified_name 映射
  ts_extract_skeleton()     — 摘要服务用的文件骨架
  format_signature()        — 语言感知的签名格式化
Rust 特定:
  _extract_rust_imports()   — 解析 use 声明
  _resolve_rust_imports()   — crate::/super::/self:: 路径解析
  _detect_rust_entries()    — fn main / #[tokio::main] / 路由属性
  _format_rust_signature()  — fn / async fn 格式
register_language("rust", _RUST_CONFIG)
```

### Rust → SymbolDef 映射

| Rust 构造 | kind | name 格式 |
|---|---|---|
| `fn foo()` | `"function"` | `"foo"` |
| `async fn foo()` | `"async_function"` | `"foo"` |
| `struct/enum/trait Foo` | `"class"` | `"Foo"` |
| `impl Foo { fn bar() }` | `"method"` | `"Foo.bar"` |
| `#[derive(...)]` | → decorators | `"#[derive(...)]"` |
| `/// doc comment` | → docstring | 去前缀后的文本 |
